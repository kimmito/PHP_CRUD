# PHP CRUD

Учебный проект: frontend на React (Vite), backend на PHP (REST API), база данных MySQL.

## Что реализовано

- CRUD для отделов, товаров и продаж.
- Формирование ведомости выручки.
- Регистрация пользователей с серверной валидацией.
- Авторизация через PHP sessions.
- Функция "Запомнить меня" через cookie с токеном.
- Разграничение прав доступа по ролям.
- Модерация заявок администратором.
- Учет количества входов и даты предыдущего входа.
- Logout с очисткой session и remember-cookie.

## Роли

- Гость: главная страница, регистрация, авторизация.
- Оператор: главная страница, ведомость, сообщение "Режим оператора активен".
- Администратор: ведомость, CRUD, модерация заявок, назначение ролей.

Новые пользователи после регистрации получают статус `pending`. Администратор одобряет заявку и назначает роль оператора.

## Стартовый администратор

После импорта `db_trade_crud.sql` доступен администратор:

- login: `admin`
- email: `admin@example.com`
- password: `admin12345`

## Запуск

1. Создайте БД `TradeDB` и импортируйте `db_trade_crud.sql`.
2. Скопируйте `api/.env.example` в `api/.env`, если используете свои параметры БД.
3. Скопируйте `frontend/.env.example` в `frontend/.env`, если API доступен не по `http://localhost/api`.
4. Установите зависимости frontend:

```bash
cd frontend
npm install
```

5. Запустите frontend:

```bash
npm run dev
```

6. Разместите папку `api` в OpenServer/XAMPP так, чтобы она открывалась по адресу `http://localhost/api`.

## Проверка

- Гость видит только главную страницу с формами регистрации и входа.
- Новый пользователь после регистрации не получает доступ до одобрения.
- Администратор открывает раздел "Пользователи" и одобряет заявку как оператора.
- Оператор видит ведомость и не имеет доступа к CRUD/админским страницам.
- Прямой ввод URL защищенных разделов блокируется на frontend и backend.
